# Contributing to TFSC's Edventurefy & EI Open Source  
  
Thanks for contributing.  
  
This project is documentation and example-first.  
The goal is to help people use/pilot/test, adapt, and improve Edventurefy and EI in the real world.  
  
## What kinds of contributions are welcome  
  
We welcome practical improvements, especially:  
  
- documentation fixes  
- broken link fixes  
- clearer language  
- better onboarding  
- improved examples  
- facilitator support improvements  
- adaptations for new audiences or contexts  
- lightweight tools, scripts, or workflows  
- implementation notes from real-world pilots  
  
## Before you contribute  
  
Please read:  
  
- `README.md`  
- `LICENSE`  
- `LICENSE-content.md`  
- `NOTICE.md`  
- `CODE_OF_CONDUCT.md`  
  
## Contribution principles  
  
Contributions should be:  
  
- practical  
- clear  
- modular  
- easy to use  
- easy to adapt  
- aligned with the mission  
  
Please avoid contributions that are:  
  
- bloated  
- overly theoretical  
- dependent on heavy infrastructure  
- unclear about purpose  
- misaligned with the mission  
  
## How to contribute  
  
### Small changes  
1. Fork the repository  
2. Make your change  
3. Submit a pull request with a clear description  
  
### Bigger changes  
1. Open an issue first  
2. Explain what you want to add or change  
3. Explain why it matters  
4. Wait for discussion before building too far ahead  
  
## What to include in your pull request  
  
Please include:  
  
- what changed  
- why it changed  
- who it helps  
- any tradeoffs or open questions  
- examples, if relevant
